# Client Delivery

## Nome

Client Delivery

## Descricao

Use esta skill para preparar a entrega final profissional do site institucional em formato de portfolio comercial para uma empresa especialista em camas e cabeceiras personalizadas.

## Quando usar

Use no final do desenvolvimento, antes da publicacao, antes de enviar para o cliente, ao criar README, ao revisar checklists ou ao documentar como manter o site.

## Instrucoes detalhadas

- Gerar ou revisar README.
- Criar checklist tecnico.
- Criar checklist visual.
- Criar checklist de seguranca.
- Criar checklist de SEO.
- Documentar como trocar imagens.
- Documentar como trocar textos.
- Documentar como alterar telefone e mensagem do WhatsApp.
- Documentar como publicar nova versao na Vercel.
- Listar o que esta incluso no projeto.
- Listar o que nao esta incluso.
- Listar melhorias futuras recomendadas.
- Escrever documentacao clara para cliente real e tambem util para manutencao tecnica.
- Referenciar caminhos reais de arquivos quando o projeto existir.

## Regras do projeto

- Considerar stack React, TypeScript, Vite, TailwindCSS e Vercel.
- Considerar site institucional sem backend.
- Considerar site sem login.
- Considerar site sem banco de dados.
- Considerar site sem upload de arquivos.
- Considerar captacao via WhatsApp.
- Considerar portfolio de camas, cabeceiras, recamiers, puffs e estofados personalizados.
- Nao documentar recursos que nao existem como se estivessem inclusos.
- Nao incluir credenciais, tokens ou informacoes privadas no README.
- Nao sugerir coleta desnecessaria de dados pessoais.

## Formato esperado da resposta

Ao finalizar, retornar:

1. Checklist final de entrega.
2. Documentacao criada ou revisada.
3. Pendencias antes da entrega.
4. Melhorias futuras recomendadas.
